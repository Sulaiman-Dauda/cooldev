# CoolDev Docs

CoolDev is built for self-hosted teams that want a clean deployment experience without stitching together a control plane by hand.

Start here if you are evaluating, installing, or operating CoolDev.

## Start here

- [Getting started](getting-started.md): product overview, what CoolDev is for, and the fastest path to value.
- [Installation](installation.md): installer flow, pinned releases, and advanced install options.
- [First deploy](first-deploy.md): the first run through owner setup, New Resource, and shipping your first workload.

## Core workflows

- [Resources](resources.md): what CoolDev can deploy and how each resource type fits.
- [Domains and HTTPS](domains-and-https.md): bootstrap URL, custom domains, DNS, and automatic HTTPS.
- [Providers](providers.md): GitHub App setup, SSH keys, and webhook behavior.
- [Deployments](deployments.md): deployment history, queue visibility, diagnostics, and logs.

## Operations

- [Self-hosting and operations](self-hosting.md): runtime expectations, release assets, validation, and operator guidance.

## Recommended path

1. Install CoolDev on a fresh server.
2. Open the bootstrap URL.
3. Create the first owner account.
4. Deploy your first resource.
5. Add a custom domain when DNS is ready.

If you want the shortest route, begin with [Getting started](getting-started.md).
